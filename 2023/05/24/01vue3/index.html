Vue2 与 Vue3 的区别

1. 挂载 App

从原来的 new Vue 到现在的按需引入,减少了引入数量,加快了项目构建

2. template 中的根节点

vue2 中只支持一个根节点,但是 vue3 支持多个根节点

script 节点

向下兼容,支持 vue2 中 data,method 的写法

全局注册组件

在 main.js 中

vue2

注册时的 name,可以使用:

  1. 短横线命名法
  2. 大驼峰命名法 (推荐)

而且其中大驼峰可以转化为短横线命名法

app.component(Movie.name,Movie),如果再创建 VC 实例时,添加了 name 属性,直接当作对象调用

scoped

css 的样式范围生效,当父组件希望子组件也生效,使用 :deep() 修饰,深度选择器

Props

自定义属性,父组件通过 props 传递数值,子类没有的被忽略

<MyCom data1="1000" :data2="1000">

注意: data1 没使用 v-bind,传递的直接为字符串,但是 data2 使用了 v-bind,所以解析为数字 1000

两种模式

  1. 数组模式 props;['data1','data2']
  2. 对象模式,可以进一步约束对象的性质,默认值…

validator

自定义 props 属性的验证函数,返回值为 Boolean 类型

Commputed(计算属性)

监听组件的变量,并作出相应处理,函数返回值保存在函数名中,可直接调用

计算属性侧重于得到一个计算的结果,因此 计算属性必须要有return 返回值

注意

自定义事件

子向父传值

基本流程

  1. 子组件定义 emits:[‘xxx’]数组,表明自定义事件,使用 this.$emit 触发
  2. 父组件通过 v-on(@)监听事件触发

区别:多了在 emits 数组中声明(不写也可以)

组件上同步数据

分为 父向子:v-bind | 子向父:v-model…

父向子: v-bind

子向父: 父组件 v-model,子组件 emits:[‘update: number’],再定义方法使用 this.$emit(‘update: number’,值),会同步修改父组件的值

优点:简化了父组件定义函数,只需要子组件定义$emit,emits

watch 监听器

监听的数据,是否在初始化时监听

deep

如果 watch 监听的是一个 对象,开启后会监听内部成员的变化,只要有任何一个值改变,就执行

handler 函数.

只监听对象的指定属性

对于 deep 监听,一个属性改变,就会执行 handler,但可能不是所期望的属性,所以指定属性监听
