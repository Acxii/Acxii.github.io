模块化

三种导出

1.默认导出

导出: export defalut 成员 每个模块只能导出 一次

导入: import Name from 'PathName',可以使用任何 Name,只要符合规范

2.按需导出

导出:export 导出的成员,每个模块可以多次

导入:import {相同的名字} from 'pathName',表示导出的成员,名称必须一致

两者配合

import A,{a,b} from 'pathName'

3.直接导入并执行模块中的代码

模块中只是相关方法,只想执行一次,直接import 'pathName',代表只执行代码

Promise

回调地狱:多层回调函数的相互嵌套

promise

使用 then-fs 对象 then()链式调用

catch()捕获错误

类似 java 的 try..catch()用于捕获错误

Promise.all()

Promise.race()

async/await

如果某个函数的返回值为 Promise 实例,则可以使用 await 获取其中的值,使用了 await,其方法必须要用 async 修饰

注意事项

EventLoop(事件循环)

js 是一门单线程的语言,如果前一个任务耗时,则后续任务不会被执行

同步任务

异步任务

JS 中同步与异步的执行顺序

  1. 同步任务 js 直接执行并相应处理.遇到异步任务给宿主环境
  2. 已经完成的异步任务对应的回调函数会放入任务队列,等待主线程同步任务执行完毕,再从任务队列中执行回调
  3. 直到任务队列为空

宏任务

对于异步任务的进一步划分

宏任务与微任务的执行顺序
